# 2048 玩法设计文档

## 1. 游戏简介
2048 是一款风靡全球的数字拼图滑块游戏。玩家需要在一个 4×4 的方格上滑动数字方块，相同数字的方块在碰撞时会相加合并成一个原来两倍大小的数字方块。游戏的核心目标是通过不断合并，最终拼凑出数字为 2048 的方块。

## 2. 核心元素
- **游戏棋盘 (Grid)**：一个 4×4 大小的正方形网格，总共 16 个格子。
- **数字方块 (Tiles)**：每个方块上带有一个数字，且均为 2 的幂次方（2, 4, 8, 16, 32, ..., 2048）。
- **生成机制 (Spawn)**：每次有效滑动后，系统会在棋盘的随机空位上生成一个新的数字方块（90% 概率生成 2，10% 概率生成 4）。

## 3. 游戏规则与操作
### 3.1 基础操作
- **PC端 (键盘)**：使用 `上(↑)`、`下(↓)`、`左(←)`、`右(→)` 方向键来控制所有方块的滑动方向。
- **移动端 (触摸)**：在屏幕上向四个方向进行滑动（Swipe）来控制。
- 每次滑动，棋盘上所有的方块都会向指定方向尽可能远地滑动，直到碰到边缘或其他方块。

### 3.2 合并与得分机制
- 当两个带有相同数字的方块在滑动过程中碰撞时，它们会合并成一个新的方块，数字为原来两者的和（如 2+2=4，4+4=8）。
- **单次滑动限制**：在同一次滑动中，一个已经由合并产生的新方块不能再次参与合并。例如一行是 `2 2 4 8` 向左滑，结果会变成 `4 4 8 0`，而不是 `8 8 0 0` 或 `16 0 0 0`。
- **得分计算**：每次成功合并两个方块，玩家会获得等同于新生成方块数字的分数（例如合并出 8，得 8 分）。总分为所有合并得分的累加。

### 3.3 游戏结束与胜利判定
- **胜利 (Win)**：当棋盘上第一次出现数字为 **2048** 的方块时，玩家获得胜利！胜利后，玩家可以选择继续游戏（Keep Going）以挑战更高的分数和更大的数字（如 4096, 8192 等）。
- **失败 (Game Over)**：当棋盘的 16 个格子全部被填满，且四个方向上均无法进行任何有效的滑动或合并时，游戏结束。

## 4. UI 界面设计
- **头部信息 (Header)**：
  - **游戏标题**：显眼的 "2048" 字样。
  - **计分板**：显示“当前分数 (Score)”和“历史最高分 (Best)”。最高分数据需存储在本地 (localStorage) 中。
  - **重新开始按钮 (New Game)**：随时可以点击重置棋盘和当前分数。
- **主游戏区 (Game Board)**：
  - 居中的 4x4 圆角网格，背景色为深灰偏棕。
  - 方块根据数字大小具有不同的背景颜色和字体颜色，数字越大颜色越深/越醒目，带有平滑的 CSS 出现动画效果。
- **操作说明 (Instructions)**：在底部简要提示“使用方向键或滑动屏幕来合并数字”。

## 5. 技术实现思路
- **DOM 渲染**：使用 HTML5 结合 CSS Grid 布局构建棋盘和方块。利用 `data-value` 属性来通过 CSS 动态设置每个方块的颜色样式。
- **数据结构**：使用一个 `4x4` 的二维数组 `grid[4][4]` 来表示当前棋盘状态。
- **滑动算法 (Slide & Merge Algorithm)**：
  - 对每一行（或列）进行操作。
  - 先将所有的非 0 元素向指定方向“压缩”（去除中间的 0）。
  - 然后遍历相邻元素，如果相等则合并，将合并结果放在前一个位置，后一个位置设为 0，同时增加得分。
  - 最后再进行一次“压缩”，确保合并后产生的空隙被填补。
- **状态判断**：
  - 每次执行滑动逻辑后，比较旧的 `grid` 和新的 `grid`。如果状态发生了改变，说明是一次有效移动，此时需要生成新的随机方块 (2 或 4)。
  - 如果 `grid` 中没有 0，且任意相邻元素均不相等，则触发 Game Over 逻辑。
